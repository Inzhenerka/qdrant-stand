# Анализ данных с pandas

## Эпизод 1. Библиотеки и почему pandas

Стандартная библиотека Python хороша для общих задач, но для работы с табличными данными она избыточно низкоуровневая. Чтобы прочитать CSV, отфильтровать строки, посчитать среднее по группам — каждый раз пришлось бы писать десятки строк. Поэтому такие задачи решает специализированная библиотека **pandas**.

`pandas` импортируется и используется по принятому в индустрии соглашению — под коротким алиасом `pd`:

```python
import pandas as pd
```

Установка в проект:

```bash
uv add pandas
```

**Сценарий курса.** Дальше мы будем работать с публичным датасетом **Gapminder** — данными о ВВП на душу населения по странам и годам с 1952 по 2007. Файлы лежат разбитыми по регионам:

```
data/
├── gapminder_gdp_oceania.csv
├── gapminder_gdp_americas.csv
├── gapminder_gdp_europe.csv
├── gapminder_gdp_africa.csv
└── gapminder_gdp_asia.csv
```

В каждом файле строки — страны, столбцы — годы (`gdpPercap_1952`, `gdpPercap_1957`, …, `gdpPercap_2007`).

## Эпизод 2. Чтение табличных данных в DataFrame

Главная функция для загрузки CSV — `pd.read_csv`. Простейший вызов:

```python
data = pd.read_csv("data/gapminder_gdp_oceania.csv")
print(data)
```

По умолчанию `pandas` сделает индекс целочисленным (0, 1, 2, …). Это редко то, что нужно: естественно, чтобы строкой управляло осмысленное имя — здесь название страны. Используется аргумент `index_col`:

```python
data_oceania = pd.read_csv("data/gapminder_gdp_oceania.csv", index_col="country")
print(data_oceania)
```

Теперь индекс строк — `'Australia'`, `'New Zealand'`. К строкам по этим именам обращаться удобнее.

После загрузки полезно сразу осмотреть данные:

```python
data_oceania.info()       # типы столбцов, число пропусков, размер
data_oceania.dtypes       # типы данных
data_oceania.columns      # имена столбцов
data_oceania.shape        # (количество строк, количество столбцов)
data_oceania.describe()   # статистика по числовым столбцам
data_oceania.T            # транспонированный взгляд: годы как строки, страны как столбцы
```

`describe()` особенно полезен на старте: для каждого числового столбца выводит count, mean, std, min, max и квартили. По одному взгляду понятно, нет ли аномалий — например, отрицательного ВВП или странных всплесков.

## Эпизод 3. DataFrame и Series

В `pandas` две основные структуры:

- **`Series`** — одномерная маркированная последовательность значений. По смыслу — один столбец таблицы.
- **`DataFrame`** — двумерная таблица, набор `Series` с общим индексом. Это основной объект, с которым работают.

Обе структуры построены на NumPy и поддерживают быстрые векторные операции и аккуратную работу с пропусками (`NaN` — Not a Number).

## Эпизод 4. Доступ по позиции (`iloc`) и по метке (`loc`)

Самая частая операция — выбрать конкретные строки и столбцы.

`iloc` — выбор по **позиции** (integer location). Работает как обычная индексация массивов, индексы с нуля:

```python
data_americas = pd.read_csv("data/gapminder_gdp_americas.csv", index_col="country")

data_americas.iloc[0, 0]     # значение в первой строке и первом столбце
data_americas.iloc[0]        # вся первая строка
data_americas.iloc[:3]       # первые три строки
data_americas.iloc[0:2, 0:3] # первые 2 строки и первые 3 столбца
```

`loc` — выбор по **меткам** индекса и имён столбцов. С точки зрения смысла данных это удобнее:

```python
data_americas.loc["Argentina", "gdpPercap_1952"]   # ВВП Аргентины в 1952
data_americas.loc["Brazil"]                        # вся строка Brazil
data_americas.loc["Brazil", "gdpPercap_1952":"gdpPercap_1972"]  # подмножество годов
```

**Тонкий момент со срезами.** У `iloc` срез работает по правилам Python — конец **исключается**: `iloc[0:2]` = строки с индексами 0, 1. У `loc` срез по меткам — конец **включается**: `loc["Italy":"Poland"]` вернёт обе страны и все, что между ними.

Двоеточие `:` без значений означает «всё»:

```python
data_americas.loc[:, "gdpPercap_1952"]    # столбец 1952 для всех стран
data_americas.loc["Brazil", :]            # все годы для Brazil
```

К столбцу обычно обращаются короче — по имени:

```python
data_americas["gdpPercap_1952"]
data_americas[["gdpPercap_1952", "gdpPercap_2007"]]   # несколько столбцов
```

## Эпизод 5. Фильтрация булевыми масками

Самый мощный инструмент работы с DataFrame — **булевы маски**. Сравнение работает поэлементно и возвращает `Series`/`DataFrame` из `True`/`False`:

```python
subset = data_americas.loc["Argentina":"Bolivia", "gdpPercap_1962":"gdpPercap_1972"]
subset > 10000
# вернёт DataFrame того же размера с True/False
```

Применённая к исходному DataFrame, маска возвращает значения, где она `True`, и `NaN` там, где `False`:

```python
mask = subset > 10000
subset[mask]
```

Это удобно, потому что `NaN` игнорируются операциями вроде `mean()`, `max()`, `sum()` — то есть фильтрованную сумму считать на готовом результате.

Чаще маска строится по логике одного столбца, а отбираются целые строки:

```python
data_americas[data_americas["gdpPercap_2007"] > 20000]
data_americas[(data_americas["gdpPercap_2007"] > 20000) & (data_americas["gdpPercap_1952"] < 5000)]
```

**Важно.** При объединении условий — **обязательно круглые скобки** и операторы `&` (И), `|` (ИЛИ), `~` (НЕ). Питоновские `and / or / not` тут не работают: они не векторизованы и дадут ошибку.

## Эпизод 6. Создание и изменение столбцов

Новый столбец — простое присваивание:

```python
data_americas["gdp_change"] = data_americas["gdpPercap_2007"] - data_americas["gdpPercap_1952"]
data_americas["high_2007"] = data_americas["gdpPercap_2007"] > 15000
```

Применить функцию ко всем значениям столбца:

```python
data_americas["rounded_2007"] = data_americas["gdpPercap_2007"].apply(lambda x: round(x, 0))
```

Для строковых столбцов есть аксессор `.str` со всеми методами строк Python:

```python
df["country_upper"] = df["country"].str.upper()
df["len_name"] = df["country"].str.len()
```

## Эпизод 7. Группировка и агрегация

Главная сила `pandas` — паттерн **split-apply-combine**: сгруппировать данные по столбцу, применить функцию к каждой группе, собрать результаты обратно. Для гипотетической таблицы со столбцами `country`, `continent`, `gdp`:

```python
df.groupby("continent")["gdp"].mean()
df.groupby("continent")["gdp"].agg(["mean", "min", "max", "count"])
df.groupby(["continent", "year"])["gdp"].mean()
```

Это эквивалент `GROUP BY` из SQL. Если вы знакомы с SQL — понимание приходит мгновенно.

## Эпизод 8. Сортировка и работа с пропусками

Сортировка:

```python
data_americas.sort_values("gdpPercap_2007")
data_americas.sort_values("gdpPercap_2007", ascending=False)
data_americas.sort_index()
```

Пропуски (`NaN`) в реальных данных встречаются часто:

```python
df.isna()                                  # маска: где пропуски
df.isna().sum()                            # количество пропусков по столбцам
df.dropna()                                # удалить строки с любым пропуском
df.dropna(subset=["gdp"])                  # только если пропуск в gdp
df.fillna(0)                               # заменить все пропуски на 0
df["gdp"].fillna(df["gdp"].median())       # заменить медианой
```

Выбор стратегии — содержательное решение: иногда пропуск означает «нет данных», иногда — «нулевое значение», иногда строку с пропуском нужно вообще удалить.

## Эпизод 9. Объединение таблиц

Если данные приходят из нескольких источников:

```python
combined = pd.concat([df_americas, df_europe], ignore_index=True)   # по вертикали

merged = students.merge(grades, left_on="id", right_on="student_id")  # как SQL JOIN
```

По умолчанию `merge` делает inner join. Для других вариантов — аргумент `how`: `"left"`, `"right"`, `"outer"`.

## Эпизод 10. Запись данных

После всех преобразований результат сохраняется обратно:

```python
data_americas.to_csv("result.csv", encoding="utf-8")
data_americas.to_excel("result.xlsx")
data_americas.to_json("result.json", force_ascii=False)
```

Если индекс осмысленный (как `country` в нашем случае) — он попадёт в файл. Если нет — добавьте `index=False`, чтобы не плодить лишний столбец.

## Эпизод 11. Базовая визуализация

`pandas` интегрирован с `matplotlib`, поэтому простые графики строятся прямо из DataFrame:

```python
import matplotlib.pyplot as plt

data_oceania.loc["Australia"].plot()
plt.xlabel("Год")
plt.ylabel("ВВП на душу населения")
plt.show()

data_americas["gdpPercap_2007"].hist(bins=15)
plt.show()

data_americas[["gdpPercap_1952", "gdpPercap_2007"]].plot(kind="bar")
plt.show()
```

Для более сложной визуализации в проде применяют `seaborn` или `plotly`, но для разведочного анализа встроенных средств обычно хватает.

## Типичные ошибки начинающих

1. **`SettingWithCopyWarning`.** Появляется при попытке изменить значение в подвыборке. Решение: `df.loc[строки, столбцы] = значение` или явная копия через `.copy()`.
2. **`and / or` вместо `& | ~` в фильтрах.** Питоновские операторы не работают поэлементно. Только `& | ~`, и обязательно скобки вокруг каждого условия.
3. **Цикл `for i, row in df.iterrows()`.** Очень медленно. В большинстве случаев есть векторизованная альтернатива через `apply`, маски, `groupby` — она в десятки раз быстрее.
4. **Забыть `index_col` при чтении.** Тогда индексом становится бессмысленная нумерация 0, 1, 2, и при следующем `to_csv → read_csv` плодится лишний столбец `Unnamed: 0`.
