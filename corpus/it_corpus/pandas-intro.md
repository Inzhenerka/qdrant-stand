# Анализ данных с pandas

## Зачем этот курс

Работа с данными — повседневная задача программиста, аналитика, инженера: разобрать выгрузку из базы, посмотреть статистику по логам, очистить таблицу перед загрузкой в отчёт, сравнить два эксперимента. Excel справляется, пока строк меньше миллиона и пока ничего не нужно повторять автоматически. Дальше — pandas: библиотека для работы с табличными данными в Python.

К концу модуля вы научитесь:

- читать CSV-файлы в DataFrame и быстро осматривать их структуру,
- выбирать строки и столбцы по позиции (`iloc`) и по меткам (`loc`),
- фильтровать данные булевыми масками,
- создавать новые столбцы и применять к данным функции,
- группировать данные по столбцу и считать агрегаты,
- сортировать и обрабатывать пропуски,
- объединять таблицы по вертикали (`concat`) и по ключу (`merge`),
- сохранять результаты обратно в файл,
- строить базовые графики прямо из DataFrame.

Никакого предварительного опыта работы с данными не требуется — но желательно знать базовый Python (модуль «Введение в Python»). С Excel/Google Sheets знакомство пригодится для аналогии, но не обязательно.

## Эпизод 1. Библиотеки и почему pandas

Стандартная библиотека Python хороша для общих задач, но для работы с табличными данными она избыточно низкоуровневая. Чтобы прочитать CSV, отфильтровать строки, посчитать среднее по группам — каждый раз пришлось бы писать десятки строк цикла. Поэтому такие задачи решает специализированная библиотека **pandas**: она держит таблицы в памяти, считает агрегаты векторизованно (то есть в десятки раз быстрее обычного цикла) и сама правильно обрабатывает пропуски в данных.

**Сценарий курса.** Дальше мы будем работать с публичным датасетом **Gapminder** — данными о ВВП на душу населения по странам и годам с 1952 по 2007. Это известный открытый набор, который Hans Rosling использовал в своих знаменитых лекциях. Файлы лежат разбитыми по регионам:

```
data/
├── gapminder_gdp_oceania.csv
├── gapminder_gdp_americas.csv
├── gapminder_gdp_europe.csv
├── gapminder_gdp_africa.csv
└── gapminder_gdp_asia.csv
```

В каждом файле строки — страны, столбцы — годы (`gdpPercap_1952`, `gdpPercap_1957`, …, `gdpPercap_2007`).

## Эпизод 2. Чтение табличных данных в DataFrame

Главная функция для загрузки CSV — `pd.read_csv`. Простейший вызов:

```python
data = pd.read_csv("data/gapminder_gdp_oceania.csv")
print(data)
```

По умолчанию `pandas` сделает индекс целочисленным (0, 1, 2, …). Это редко то, что нужно: удобнее, когда роль индекса играет осмысленное имя — здесь название страны. Указывается через аргумент `index_col`:

```python
data_oceania = pd.read_csv("data/gapminder_gdp_oceania.csv", index_col="country")
print(data_oceania)
```

Теперь индекс строк — `'Australia'`, `'New Zealand'`. К строкам по этим именам обращаться удобнее.

После загрузки полезно сразу осмотреть данные:

```python
data_oceania.info()       # типы столбцов, число пропусков, размер
data_oceania.dtypes       # типы данных по столбцам
data_oceania.columns      # имена столбцов
data_oceania.shape        # (количество строк, количество столбцов)
data_oceania.head()       # первые 5 строк
data_oceania.tail()       # последние 5 строк
data_oceania.describe()   # статистика по числовым столбцам
data_oceania.T            # транспонированный взгляд: годы как строки, страны как столбцы
```

`describe()` особенно полезен на старте: для каждого числового столбца выводит count, mean, std, min, max и квартили. По одному взгляду понятно, нет ли аномалий — например, отрицательного ВВП или странных всплесков.

## Эпизод 3. DataFrame и Series

В `pandas` две основные структуры:

- **`Series`** — одномерная маркированная последовательность значений. По смыслу — один столбец таблицы.
- **`DataFrame`** — двумерная таблица, набор `Series` с общим индексом. Это основной объект, с которым работают.

Обе структуры построены на NumPy и поддерживают быстрые векторные операции и аккуратную работу с пропусками (`NaN` — Not a Number).

### Создание с нуля

Обе структуры можно собрать в коде без чтения файла — пригодится для тестов и быстрых экспериментов:

```python
# Series — из списка значений; индексом по умолчанию станут 0, 1, 2, ...
gdp_2007 = pd.Series([12779, 9065, 36126, 33203], 
                     index=["Argentina", "Bolivia", "Australia", "New Zealand"],
                     name="gdp_2007")
gdp_2007["Argentina"]    # 12779

# DataFrame — из словаря, где ключи становятся столбцами
df = pd.DataFrame({
    "country": ["Argentina", "Bolivia", "Australia"],
    "continent": ["Americas", "Americas", "Oceania"],
    "gdp_2007": [12779, 9065, 36126],
})
```

Так удобно собрать маленькую таблицу, чтобы проверить, как ведёт себя ваш код, не открывая большой файл.

### Извлечение столбца как Series

Любой столбец DataFrame — это Series:

```python
australia_series = data_oceania.loc["Australia"]    # тип: Series
type(australia_series)    # <class 'pandas.core.series.Series'>
```

Это удобно, когда вы хотите дальше работать с одним столбцом отдельно — например, считать статистики или строить график.

## Эпизод 4. Доступ по позиции (`iloc`) и по метке (`loc`)

Самая частая операция — выбрать конкретные строки и столбцы.

`iloc` — выбор по **позиции** (integer location). Работает как обычная индексация массивов, индексы с нуля:

```python
data_americas = pd.read_csv("data/gapminder_gdp_americas.csv", index_col="country")

data_americas.iloc[0, 0]     # значение в первой строке и первом столбце
data_americas.iloc[0]        # вся первая строка
data_americas.iloc[:3]       # первые три строки
data_americas.iloc[0:2, 0:3] # первые 2 строки и первые 3 столбца
```

`loc` — выбор по **меткам** индекса и имён столбцов. С точки зрения смысла данных это удобнее:

```python
data_americas.loc["Argentina", "gdpPercap_1952"]   # ВВП Аргентины в 1952
data_americas.loc["Brazil"]                        # вся строка Brazil
data_americas.loc["Brazil", "gdpPercap_1952":"gdpPercap_1972"]  # подмножество годов
```

**Тонкий момент со срезами.** У `iloc` срез работает по правилам Python — конец **исключается**: `iloc[0:2]` = строки с индексами 0, 1. У `loc` срез по меткам — конец **включается**: `loc["Italy":"Poland"]` вернёт обе страны и все, что между ними.

Двоеточие `:` без значений означает «всё»:

```python
data_americas.loc[:, "gdpPercap_1952"]    # столбец 1952 для всех стран
data_americas.loc["Brazil", :]            # все годы для Brazil
```

К столбцу обычно обращаются короче — по имени:

```python
data_americas["gdpPercap_1952"]
data_americas[["gdpPercap_1952", "gdpPercap_2007"]]   # несколько столбцов
```

### Маска внутри loc

Частая идиома — совместить булеву маску с выбором столбцов в одном `loc`:

```python
data_americas.loc[data_americas["gdpPercap_2007"] > 20000, ["gdpPercap_1952", "gdpPercap_2007"]]
```

Эта строка читается так: «возьми строки, где ВВП в 2007 году больше 20 000, и из них покажи только два столбца — за 1952 и 2007». Так можно делать сразу и фильтрацию строк, и выбор колонок одной командой.

## Эпизод 5. Фильтрация булевыми масками

Главный инструмент фильтрации в pandas — **булевы маски**. Сравнение работает поэлементно и возвращает `Series`/`DataFrame` из `True`/`False`:

```python
subset = data_americas.loc["Argentina":"Bolivia", "gdpPercap_1962":"gdpPercap_1972"]
subset > 10000
# вернёт DataFrame того же размера с True/False
```

Применённая к исходному DataFrame, маска возвращает значения, где она `True`, и `NaN` там, где `False`:

```python
mask = subset > 10000
subset[mask]
```

Это удобно: `mean()`, `max()`, `sum()` пропускают `NaN`, и фильтрованную сумму можно посчитать прямо на таком результате.

Чаще маска строится по логике одного столбца, а отбираются целые строки:

```python
data_americas[data_americas["gdpPercap_2007"] > 20000]
data_americas[(data_americas["gdpPercap_2007"] > 20000) & (data_americas["gdpPercap_1952"] < 5000)]
```

**Важно.** При объединении условий — **обязательно круглые скобки** и операторы `&` (И), `|` (ИЛИ), `~` (НЕ). Питоновские `and / or / not` тут не работают: они не векторизованы и дадут ошибку.

### Полезные методы для масок

Кроме сравнений, маску часто строят через специальные методы:

```python
data_americas["gdpPercap_2007"].between(10000, 20000)   # 10000 <= x <= 20000
data_americas.index.isin(["Brazil", "Argentina", "Chile"])   # x в списке
data_americas["gdpPercap_2007"].notna()                 # не пропуск
```

`isin` — особенно частый случай: вместо громоздкого `(col == "A") | (col == "B") | (col == "C")` пишется одной строкой.

## Эпизод 6. Создание и изменение столбцов

Новый столбец — простое присваивание:

```python
data_americas["gdp_change"] = data_americas["gdpPercap_2007"] - data_americas["gdpPercap_1952"]
data_americas["high_2007"] = data_americas["gdpPercap_2007"] > 15000
```

Применить функцию ко всем значениям столбца — `apply`:

```python
data_americas["rounded_2007"] = data_americas["gdpPercap_2007"].apply(lambda x: round(x, 0))
```

Если нужна простая замена «значение → значение» — удобнее `map` со словарём:

```python
continent_emoji = {"Americas": "🌎", "Europe": "🌍", "Asia": "🌏", "Africa": "🌍", "Oceania": "🌏"}
df["icon"] = df["continent"].map(continent_emoji)
```

`map` короче и быстрее `apply`, когда отображение задаётся справочником.

Для строковых столбцов есть аксессор `.str` со всеми методами строк Python:

```python
df["country_upper"] = df["country"].str.upper()
df["len_name"] = df["country"].str.len()
df["starts_with_A"] = df["country"].str.startswith("A")
```

## Эпизод 7. Группировка и агрегация

Группировка в pandas построена на паттерне **split-apply-combine**: разбить данные по столбцу, применить функцию к каждой группе, склеить результаты обратно. Для гипотетической таблицы со столбцами `country`, `continent`, `gdp`:

```python
df.groupby("continent")["gdp"].mean()
df.groupby("continent")["gdp"].agg(["mean", "min", "max", "count"])
df.groupby(["continent", "year"])["gdp"].mean()
```

Это эквивалент `GROUP BY` из SQL. Если вы знакомы с SQL — понимание приходит мгновенно. Если нет — посмотрите модуль про SQL в этом же курсе: оператор `GROUP BY` устроен на той же идее.

`agg` принимает список функций или словарь «столбец → функция». Можно задать кастомные:

```python
df.groupby("continent").agg(
    gdp_mean=("gdp", "mean"),
    countries=("country", "nunique"),
)
```

## Эпизод 8. Сортировка и работа с пропусками

Сортировка:

```python
data_americas.sort_values("gdpPercap_2007")
data_americas.sort_values("gdpPercap_2007", ascending=False)
data_americas.sort_index()
```

Пропуски (`NaN`) в реальных данных встречаются почти всегда. С каждым пропуском нужно явно решить, что с ним делать:

```python
df.isna()                                  # маска: где пропуски
df.isna().sum()                            # количество пропусков по столбцам
df.dropna()                                # удалить строки с любым пропуском
df.dropna(subset=["gdp"])                  # только если пропуск в gdp
df.fillna(0)                               # заменить все пропуски на 0
df["gdp"].fillna(df["gdp"].median())       # заменить медианой
```

Выбор стратегии — содержательное решение: иногда пропуск означает «нет данных», иногда — «нулевое значение», иногда строку с пропуском нужно вообще удалить. Это вопрос предметной области, а не технический выбор.

## Эпизод 9. Объединение таблиц

Если данные приходят из нескольких источников, есть два основных способа их склеить.

**`concat`** — приклеить таблицы друг к другу. Используется, когда у вас несколько кусков одних и тех же данных:

```python
combined = pd.concat([df_americas, df_europe], ignore_index=True)   # по вертикали (одинаковые столбцы)
side_by_side = pd.concat([df_a, df_b], axis=1)                       # по горизонтали (одинаковые строки)
```

**`merge`** — соединить две таблицы по общему ключу, как `JOIN` в SQL. Используется, когда у каждой таблицы свой набор столбцов, и их нужно связать:

```python
students.merge(grades, on="student_id")                            # общий столбец называется одинаково
students.merge(grades, left_on="id", right_on="student_id")        # имена столбцов разные
```

По умолчанию `merge` делает inner join — оставляет только строки, ключ которых есть в обеих таблицах. Для других режимов — аргумент `how`:

- `how="left"` — все строки левой таблицы; правая, где нет соответствия, даст `NaN`.
- `how="right"` — наоборот.
- `how="outer"` — все строки обеих таблиц, недостающее — `NaN`.

В реальных задачах `left join` встречается чаще всего: «у меня есть таблица студентов, и я хочу подтянуть оценки тех, у кого они есть, а остальных оставить с пустыми».

## Эпизод 10. Запись данных

После всех преобразований результат сохраняется обратно:

```python
data_americas.to_csv("result.csv", encoding="utf-8")
data_americas.to_excel("result.xlsx")
data_americas.to_json("result.json", force_ascii=False)
```

Если индекс осмысленный (как `country` в нашем случае) — он попадёт в файл. Если нет — добавьте `index=False`, чтобы не плодить лишний столбец:

```python
df.to_csv("result.csv", index=False)
```

Иначе при следующем чтении файл вырастет на один технический столбец `Unnamed: 0`, и так с каждой итерацией.

## Эпизод 11. Базовая визуализация

`pandas` интегрирован с `matplotlib`, поэтому простые графики строятся прямо из DataFrame:

```python
import matplotlib.pyplot as plt

# Линейный график динамики ВВП Австралии по годам
data_oceania.loc["Australia"].plot()
plt.xlabel("Год")
plt.ylabel("ВВП на душу населения")
plt.title("ВВП Австралии, 1952–2007")
plt.show()

# Гистограмма распределения ВВП в 2007 году
data_americas["gdpPercap_2007"].hist(bins=15)
plt.show()

# Столбчатая диаграмма: ВВП в 1952 vs 2007
data_americas[["gdpPercap_1952", "gdpPercap_2007"]].plot(kind="bar")
plt.show()

# Scatter: связь между ВВП 1952 и 2007
data_americas.plot(kind="scatter", x="gdpPercap_1952", y="gdpPercap_2007")
plt.show()
```

Для более сложной визуализации в проде применяют `seaborn` (статистические графики) или `plotly` (интерактивные), но для разведочного анализа встроенных средств обычно хватает.

## Типичные ошибки начинающих

1. **`SettingWithCopyWarning`.** Появляется при попытке изменить значение в подвыборке (`df[df["x"] > 0]["y"] = 1`). Решение: всегда `df.loc[строки, столбцы] = значение` или явная копия через `.copy()`.
2. **`and / or` вместо `& | ~` в фильтрах.** Питоновские операторы не работают поэлементно. Только `& | ~`, и обязательно скобки вокруг каждого условия.
3. **Цикл `for i, row in df.iterrows()`.** Очень медленно. В большинстве случаев есть векторизованная альтернатива через `apply`, маски, `groupby` — она в десятки раз быстрее. Цикл по DataFrame — это «code smell».
4. **Забыть `index_col` при чтении.** Тогда индексом становится бессмысленная нумерация 0, 1, 2, и при следующем `to_csv → read_csv` плодится лишний столбец `Unnamed: 0`.
5. **Использовать `==` для сравнения с `NaN`.** `NaN == NaN` это `False` — такова семантика плавающей запятой. Для проверки пропуска есть `df.isna()` / `df.notna()`.

## Что дальше

К этому моменту вы умеете:

- ставить pandas и импортировать его под алиасом `pd`,
- читать CSV в DataFrame, ставить осмысленный индекс через `index_col`, быстро осматривать таблицу через `info`/`describe`,

Этого хватает для повседневной аналитической работы. Следующий логичный шаг — научиться писать **автоматические тесты** для своих функций обработки данных, чтобы изменения в коде не ломали аналитику. Этим занимается следующий модуль курса.
