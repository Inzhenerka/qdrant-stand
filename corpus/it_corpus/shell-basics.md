# Основы Unix Shell

## Эпизод 1. Зачем нужен shell

С компьютером можно общаться двумя способами:

- **Графический интерфейс (GUI)** — окна, кнопки, мышь. Удобно для повседневных задач: открыть файл, отправить письмо, посмотреть видео.
- **Командная строка (CLI)** — текстовое окно, в которое вводятся команды. Менее интуитивно, но **многократно эффективнее** для повторяющихся задач, автоматизации и работы с большими объёмами файлов.

**Shell** — это программа, которая принимает текстовые команды и передаёт их операционной системе на выполнение. На этом курсе используется **Bash** (Bourne Again SHell) — самая распространённая Unix-оболочка. На macOS по умолчанию стоит Zsh, очень похожий на Bash. На Windows для работы с Bash удобнее всего поставить **Git Bash** или WSL.

**Сценарий курса.** Морской биолог Nelle Nemo вернулась из экспедиции в район North Pacific Gyre с **1520 образцами** морской живности. Каждый образец нужно прогнать через программу `goostats.sh` для статистического анализа. Делать это вручную через GUI, открывая каждый файл по одному, — около 12 часов рутины. С помощью shell ту же работу можно описать одной командой и получить результат за минуты.

При запуске Bash показывает приглашение (prompt) — обычно `$` для обычного пользователя или `#` для root. После приглашения вы пишете команду, нажимаете Enter, shell её выполняет и снова показывает приглашение.

Самые первые команды, чтобы убедиться, что всё работает:

```bash
$ whoami     # ваше имя пользователя
$ ls         # содержимое текущей папки
```

## Эпизод 2. Навигация по файловой системе

В любой момент времени вы находитесь в одной конкретной папке — она называется **рабочей директорией** (working directory). Узнать, где вы:

```bash
$ pwd
/home/nelle
```

Содержимое папки:

```bash
$ ls
Desktop  Documents  Downloads  shell-lesson-data
```

Полезные опции `ls`:

- `ls -F` — добавляет к именам символы-подсказки: `/` для директорий, `*` для исполняемых файлов.
- `ls -l` — подробный формат: права, владелец, размер, дата изменения.
- `ls -a` — включая скрытые файлы (имена начинаются с точки).
- `ls -h` — размеры в человекочитаемом виде (5.3K вместо 5369).

Опции можно комбинировать: `ls -lah`.

Перейти в другую папку — команда `cd`:

```bash
$ cd Desktop
$ cd shell-lesson-data
$ pwd
/home/nelle/Desktop/shell-lesson-data
```

Специальные обозначения:

- `..` — родительская директория.
- `.` — текущая директория.
- `~` — домашняя директория (`/home/nelle`).
- `cd` без аргументов — тоже в домашнюю.
- `cd -` — вернуться в предыдущую директорию.

Пути бывают двух видов:

- **Абсолютный** начинается с `/` и описывает полный маршрут от корня файловой системы: `/home/nelle/Desktop/shell-lesson-data`.
- **Относительный** задаётся от текущей директории: `Desktop/shell-lesson-data` или `../Documents`.

В курсе мы будем работать в учебных данных Nelle, лежащих в:

```
~/Desktop/shell-lesson-data/
├── north-pacific-gyre/          ← 1520 файлов вида NENE01729A.txt
└── exercise-data/
    ├── alkanes/                 ← cubane.pdb, ethane.pdb, methane.pdb…
    ├── animal-counts/           ← animals.txt со списком встреченных видов
    ├── creatures/
    ├── numbers.txt
    └── writing/                 ← haiku.txt и другие текстовые файлы
```

**Tab-автодополнение.** Начните вводить имя папки или файла и нажмите Tab — оболочка дополнит имя, если вариант однозначен. Это резко ускоряет навигацию.

## Эпизод 3. Создание, копирование, удаление

Создать пустую папку:

```bash
$ mkdir thesis                                # одну
$ mkdir -p ../project/data ../project/results # с цепочкой родителей
```

Создать пустой файл (или обновить дату изменения существующего):

```bash
$ touch draft.txt
```

Открыть файл в простом текстовом редакторе для правки:

```bash
$ nano draft.txt
```

В `nano` основные сочетания: `Ctrl+O` сохранить, `Ctrl+X` выйти.

Переименовать или переместить (это одна и та же операция в Unix):

```bash
$ mv thesis/draft.txt thesis/quotes.txt   # переименовать внутри папки
$ mv thesis/quotes.txt .                  # перенести в текущую папку
$ mv -i quotes.txt notes.txt              # с подтверждением перезаписи
```

Скопировать файл:

```bash
$ cp quotes.txt thesis/quotations.txt     # один файл
$ cp -r thesis thesis_backup              # папку рекурсивно (флаг -r)
```

Удалить файл или папку:

```bash
$ rm quotes.txt
$ rm -r thesis             # папку со всем содержимым
$ rm -i thesis_backup/*    # с подтверждением каждого файла
```

**Важнейшее предостережение.** В Unix shell **нет «корзины»**. Команда `rm` удаляет файл сразу и навсегда. Это не «переместить в Trash» — это безвозвратное удаление. Особенно опасна `rm -rf путь` — она молча удаляет директорию со всем содержимым, без вопросов. Никогда не запускайте `rm -rf` вслепую и тем более под суперпользователем — одна неверная буква в пути может стоить вам всех данных.

## Эпизод 4. Перенаправление и пайпы

В директории `~/Desktop/shell-lesson-data/exercise-data/alkanes/` лежат шесть файлов с описаниями молекул в формате PDB: `cubane.pdb`, `ethane.pdb`, `methane.pdb`, `octane.pdb`, `pentane.pdb`, `propane.pdb`.

**Подсчёт.** Команда `wc` (word count) считает строки, слова и символы:

```bash
$ wc cubane.pdb
20  156  1158 cubane.pdb
$ wc -l *.pdb       # только строки во всех .pdb
```

**Содержимое.** Команда `cat` выводит файл целиком; для длинных файлов удобнее `less` (постраничный просмотрщик с навигацией стрелками и `q` для выхода). Первые/последние строки:

```bash
$ head pentane.pdb        # первые 10 строк
$ head -n 5 pentane.pdb   # первые 5
$ tail -n 20 pentane.pdb  # последние 20
```

**Сортировка.** Команда `sort` по умолчанию сортирует строки лексикографически; флаг `-n` — численно:

```bash
$ sort numbers.txt
$ sort -n numbers.txt
$ sort -rn numbers.txt    # численно по убыванию
```

**Уникальные значения.** `uniq` оставляет только уникальные **подряд идущие** строки (поэтому почти всегда применяется после `sort`); `-c` выводит количество повторов:

```bash
$ sort animal-counts/animals.txt | uniq
$ sort animal-counts/animals.txt | uniq -c
```

**Перенаправление.** Стандартный вывод команды можно записать в файл:

```bash
$ wc -l *.pdb > line_counts.txt    # перезаписать
$ wc -l *.pdb >> line_counts.txt   # дописать в конец
```

Стандартный ввод — взять из файла:

```bash
$ sort < numbers.txt
```

**Пайпы.** Самая мощная конструкция Unix — символ `|`, который передаёт вывод одной команды на вход другой. Так сложные задачи собираются из простых утилит. Например, найти PDB-файл с наименьшим числом строк:

```bash
$ wc -l *.pdb | sort -n | head -n 1
```

Эта цепочка читается слева направо: посчитать строки во всех `.pdb`, отсортировать численно по возрастанию, взять верхнюю строку (с минимумом).

Это и есть философия Unix: **простые инструменты, каждый делает одну задачу хорошо и хорошо стыкуется с другими**.

## Эпизод 5. Циклы

Когда одно и то же действие нужно повторить для нескольких файлов, помогает цикл `for`:

```bash
$ for filename in *.pdb
> do
>     echo "Обрабатываю $filename"
>     head -n 1 "$filename"
> done
```

Переменная цикла (здесь `filename`) поочерёдно принимает значения из списка после `in`. Внутри тела цикла к ней обращаются через `$filename`. Кавычки вокруг `"$filename"` важны — они защищают от пробелов в именах файлов.

Полезный приём — переименовать сразу пачку файлов, добавив префикс:

```bash
$ for f in *.pdb
> do
>     mv "$f" "original-$f"
> done
```

В сценарии Nelle ровно так автоматизируется обработка 1520 файлов: пишется одна команда с циклом, и за минуты делается то, что вручную заняло бы день.

## Эпизод 6. Shell-скрипты

Если последовательность команд нужно использовать многократно, её сохраняют в файл — это и есть **shell-скрипт**. Скрипт — это маленькая программа.

Создаём `middle.sh`, который вырезает середину файла:

```bash
# middle.sh — выводит N строк из середины файла
head -n "$2" "$1" | tail -n "$3"
```

Здесь `$1`, `$2`, `$3` — это первый, второй, третий аргумент при запуске скрипта. Запуск:

```bash
$ bash middle.sh octane.pdb 15 5
```

Что произойдёт: возьмутся первые 15 строк `octane.pdb`, а из них — последние 5. То есть строки с 11-й по 15-ю. Параметризованная утилита для просмотра середины любого файла.

Ещё более универсальный вариант — `sorted.sh`, сортирующий любое количество файлов по числу строк:

```bash
# sorted.sh — сортирует переданные файлы по количеству строк
wc -l "$@" | sort -n
```

`$@` — это **все** аргументы командной строки разом. Запуск:

```bash
$ bash sorted.sh *.pdb
$ bash sorted.sh ../../alkanes/*.pdb ../numbers.txt
```

**Хорошая практика.**

- В начале скрипта добавляйте комментарий-описание (строки с `#`): что делает скрипт и какие у него аргументы.
- Заключайте `"$1"`, `"$@"` в двойные кавычки — это защита от пробелов и спецсимволов в путях.
- Если хотите запускать скрипт без явного `bash` — добавьте первой строкой shebang `#!/bin/bash` и сделайте файл исполняемым через `chmod +x middle.sh`. Тогда запуск — просто `./middle.sh octane.pdb 15 5`.

Скрипты — главный инструмент **воспроизводимости**. Если вы автоматизировали обработку данных скриптом, эту обработку легко повторить, проверить, передать коллеге, поставить под контроль версий через Git.

## Эпизод 7. Поиск файлов и текста

В большом проекте нужно уметь быстро находить файлы по имени и строки по содержимому.

**Поиск строк в файлах: `grep`.** Команда ищет строки, соответствующие шаблону, в одном или нескольких файлах. Пусть в папке `writing/` лежит файл `haiku.txt`. Найдём строки со словом «the»:

```bash
$ grep "the" haiku.txt
```

Полезные опции:

- `-n` — показать номера строк.
- `-w` — искать только целое слово (а не подстроку).
- `-i` — игнорировать регистр.
- `-v` — наоборот, показать строки, **не содержащие** шаблон.
- `-r` — рекурсивный поиск во всех файлах дерева.
- `-E` — использовать расширенные регулярные выражения (доступны `+`, `?`, `|`).

Несколько примеров:

```bash
$ grep -n -w -i "the" haiku.txt        # все вхождения слова "the" с номерами строк
$ grep -rn "TODO" .                    # все TODO в текущем дереве
$ grep -E "cat|dog" animals.txt        # либо cat, либо dog
```

**Поиск файлов: `find`.** Ищет файлы и папки по разным критериям, рекурсивно:

```bash
$ find . -type f                       # все файлы в дереве
$ find . -type d                       # только директории
$ find . -name "*.txt"                 # все .txt в дереве
$ find . -name "haiku.*"               # haiku любого расширения
$ find . -mtime -7                     # изменённые за последние 7 дней
$ find . -size +10M                    # файлы больше 10 МБ
```

**Важно:** wildcards (`*`, `?`) в `-name` нужно **заключать в кавычки**, иначе их раскроет сама оболочка ещё до запуска `find`, и вы получите неожиданное поведение.

**Комбинация `find` и других команд.** Самый мощный приём — подставить вывод `find` как аргументы другой команды через `$(...)`:

```bash
$ wc -l $(find . -name "*.txt")            # посчитать строки во всех .txt
$ grep "searching" $(find . -name "*.txt") # искать "searching" во всех .txt
```

Альтернативный синтаксис через `xargs` (надёжнее при большом количестве файлов):

```bash
$ find . -name "*.txt" | xargs wc -l
$ find . -name "*.txt" | xargs grep "searching"
```

Так за одну команду проходится дерево из тысяч файлов — то, ради чего Nelle и взялась за shell.
